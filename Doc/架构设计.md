# NewLife.Core 架构设计

> 版本：v11.17 | 日期：2026-07-15 | 从代码结构逆向整理

本文档描述 NewLife.Core 核心库的架构设计。需求定义见[需求文档](需求文档.md)，功能追踪见[功能清单](功能清单.md)。

## 1. 项目分层

NewLife.Core 采用**单项目 + 目录命名空间**的轻量分层架构，不拆分多项目（符合 NewLife 从两层起步的理念）。

```
NewLife.Core/
├── Common/          # 根命名空间 NewLife.*（基础类型）
├── Extension/       # NewLife.* 扩展方法
├── Event/           # NewLife.* 事件与弱引用委托
├── Exceptions/      # NewLife.* 异常类型
├── Stub/            # 兼容性填充（低版本 .NET 缺失类型）
├── Collections/     # NewLife.Collections 集合与池化
├── Caching/         # NewLife.Caching 缓存系统
├── Configuration/   # NewLife.Configuration 配置系统
├── Data/            # NewLife.Data 数据结构
├── Log/             # NewLife.Log 日志与追踪
├── Model/           # NewLife.Model 应用框架
├── Reflection/      # NewLife.Reflection 反射扩展
├── Net/             # NewLife.Net 网络库
│   └── Handlers/    # 编解码器
├── Http/            # NewLife.Http HTTP 组件
├── Remoting/        # NewLife.Remoting RPC 框架
├── Serialization/   # NewLife.Serialization 序列化
│   ├── Json/        # JSON 序列化子模块
│   └── Xml/         # XML 序列化子模块
├── Security/        # NewLife.Security 安全加密
├── Threading/       # NewLife.Threading 定时调度
├── Messaging/       # NewLife.Messaging 消息与事件
├── Web/             # NewLife.Web Web 工具
├── IO/              # NewLife.IO 文件与存储
├── Compression/     # NewLife.Compression 压缩归档
├── Algorithms/      # NewLife.Algorithms 数据算法
├── Buffers/         # NewLife.Buffers 高性能缓冲区
├── Xml/             # NewLife.Xml XML 配置
├── Windows/         # NewLife.Windows Windows 特定
├── Setting.cs       # 核心配置 Setting
└── NewLife.Core.csproj  # 多目标框架：net45~net10.0
```

## 2. 模块架构

### 2.1 基础扩展（M1）

层内无依赖，被所有上层模块依赖。

核心组件：`Utility`（类型转换）、`StringHelper`（字符串）、`DisposeBase`（资源释放）、`Runtime`（运行时）、`TimeProvider`（时间抽象）、`IDictionarySource`（字典数据源）、`MachineInfo`（机器信息）。

## 3. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 项目结构 | 单项目 + 目录分区 | 符合两层起步理念，减少维护成本 |
| 命名空间 | 每目录一个独立命名空间 | 清晰模块边界，便于按需引用 |
| 多目标框架 | net45~net10.0（17 个 TFM） | 最大兼容性，Stub 目录填充缺失类型 |
| 日志设计 | `ILog` 接口 + `XTrace` 静态门面 | 兼顾 DI 注入和静态便捷访问 |
| 缓存设计 | `ICache` 统一接口 | 内存/Redis/自定义实现可互换 |
| 序列化 | 自研高性能实现 | 避免依赖第三方库，控制性能 |
| 网络库 | 全异步 Socket | 单机实测 2266 万 tps |
| 数据包 | `IPacket` 零拷贝切片 | 网络层核心性能优化点 |
| IoC | `ObjectContainer` 自研 | 轻量级，兼容 `IServiceProvider` |
| 负载均衡 | 故障转移/加权轮询/竞速 | 覆盖不同场景需求 |

## 5. 编码约定

见项目根目录 `.editorconfig` 和 `global.json`。

| 约定 | 说明 |
|------|------|
| 类型名 | .NET 正式名：`String`/`Int32`/`Boolean`（非 `string`/`int`/`bool`） |
| Nullable | 全局启用 `<Nullable>enable</Nullable>` |
| 语言版本 | `<LangVersion>latest</LangVersion>`（C# 14） |
| 文档注释 | XML 注释所有 public/protected 成员，`<summary>` 同行闭合 |
| 签名 | 强命名（`newlife.snk`） |
